# Оценка стоимости обогащения (enrichment) — Valencia Radar

**Задача:** T164. **Дата:** 2026-06-21.
**Связанные задачи:** T139 (model tiering), T144 (local-first bake), T162 (авторизация enrich в prod), T165 (схема крона).

Этот документ оценивает стоимость в **долларах** запуска Claude-обогащения карточек
(перевод на русский + заземлённое извлечение фактов) и подробно описывает вариант
**«prod без AI»** — когда боевой сервер не платит за AI вообще.

---

## 0. Что такое «обогащение» здесь

После того как карточка прошла детерминированный конвейер (ingest → normalize → dedup →
score → tag), её можно **обогатить** через Claude. В коде есть две дорожки (T139, см.
`lib/pipeline/enrich-client.ts`, функция `pickEnrichModel`):

| Дорожка | Что делает | Модель | Когда |
|---|---|---|---|
| **PATH A** (translation) | Перевод/перефраз `title_ru` + `description_ru` на русский, аккуратная категория | **Haiku 4.5** (дёшево) | по умолчанию (`web=false`) |
| **PATH B** (grounded) | Открывает ссылки-источники (WebFetch), читает страницу, извлекает точный адрес/часы/цену и **цитирует** их | **Sonnet 4.6** (потолок надёжности) | только `web=true` |

Движок по умолчанию — `claude -p` (CLI на **подписке/OAuth**, **без API-ключа**) →
**маржинальная стоимость $0**, запускается ЛОКАЛЬНО. Альтернатива для prod/serverless —
`@anthropic-ai/sdk` + `ANTHROPIC_API_KEY` (платит за токены). Этот документ оценивает
именно **денежную (API-ключ)** стоимость, потому что подписочный путь = $0 (входит в
фикс-плату подписки Claude).

## 1. Тарифы (авторитетные, на 1 млн токенов = MTok)

| Модель | ID | Вход (input) | Выход (output) | Контекст |
|---|---|---:|---:|---|
| **Haiku 4.5** | `claude-haiku-4-5` | **$1.00** / MTok | **$5.00** / MTok | 200K |
| **Sonnet 4.6** | `claude-sonnet-4-6` | **$3.00** / MTok | **$15.00** / MTok | 1M |

Скидки/модификаторы:
- **Prompt caching:** чтение из кэша ≈ **0.1×** input; запись в кэш **1.25×** (TTL 5 мин) / **2×** (TTL 1 ч).
- **Batch API:** **−50%** и на input, и на output (асинхронно, ≤24 ч).

## 2. Стоимость одной карточки (bottom-up)

### Допущения по токенам (заявлены явно)

| | Вход (input) | Выход (output) | Обоснование |
|---|---:|---:|---|
| **PATH A** (haiku, перевод) | **1 500 ток** | **300 ток** | системный промпт + текст карточки (заголовок + описание, обычно короткие); на выходе RU-заголовок + 1–2 предложения |
| **PATH B** (sonnet, заземление) | **10 000 ток** | **500 ток** | промпт + текст карточки + содержимое 1–4 загруженных через WebFetch страниц (основной объём — именно страница); на выходе структурный JSON с цитатами |

### Арифметика — PATH A (Haiku, перевод)

```
input :  1 500 / 1 000 000 × $1.00  = $0.0015
output:    300 / 1 000 000 × $5.00  = $0.0015
                                     ─────────
ИТОГО за карточку (PATH A)          ≈ $0.0030   (~0.3 цента)
```

### Арифметика — PATH B (Sonnet, заземление)

```
input : 10 000 / 1 000 000 × $3.00  = $0.0300
output:    500 / 1 000 000 × $15.00 = $0.0075
                                     ─────────
ИТОГО за карточку (PATH B)          ≈ $0.0375   (~3.8 цента)
```

> Итого: дорогая дорожка (B) примерно в **12 раз** дороже дешёвой (A) — поэтому tiering
> из T139 (haiku по умолчанию, sonnet только при `web=true`) экономит основную массу денег.

## 3. Объём при живой нагрузке

- **Разовый bake текущего seed:** уже сделано (T144, Tick M) — `claude -p`, $0. Для оценки в
  долларах считаем «как если бы платили по API»: **~340 карточек** (в seed сейчас 342 события,
  все с `title_ru`). Допущение раскладки: основная масса — перевод (A), небольшая доля — с
  заземлением (B). Возьмём **300 × A + 40 × B**.
- **Новые карточки/день (steady-state):** адаптивный диспетчер (T011–T015, см. §6) опрашивает
  ~25 включённых источников; реалистичный поток **новых уникальных** событий после dedup —
  **десятки в день, не тысячи**. Допущение: **30 новых карточек/день** → ~**900/мес**.
  Раскладка та же ~88% A / ~12% B → **~790 A + ~110 B** в месяц.

## 4. Три сценария (с/без Batch и с кэшем чтения)

Базовые цены за карточку: **A = $0.0030**, **B = $0.0375**.
Batch −50%: **A = $0.0015**, **B = $0.01875**.
Кэш чтения (повторяющийся системный промпт ≈ 0.1× input — влияет в основном на input-часть):
при наших долях токенов экономит ориентировочно ~**5–7%** на A и ~**1–2%** на B (на B
доминирует уникальное содержимое страницы, которое не кэшируется) — в таблице показан
ориентир «с кэшем + batch».

### Сценарий 1 — разовый полный bake текущего seed (300 A + 40 B)

| Режим | Стоимость A | Стоимость B | **ИТОГО $/прогон** |
|---|---:|---:|---:|
| Полная цена | 300×$0.0030 = $0.90 | 40×$0.0375 = $1.50 | **≈ $2.40** |
| + Batch −50% | $0.45 | $0.75 | **≈ $1.20** |
| + Batch + кэш чтения | ~$0.43 | ~$0.74 | **≈ $1.17** |

> На подписке (`claude -p`) этот прогон стоил **$0** (так и сделано). По API — около **$2** разово.

### Сценарий 2 — steady-state, инкремент новых карточек (790 A + 110 B / мес)

| Режим | Стоимость A | Стоимость B | **ИТОГО $/мес** |
|---|---:|---:|---:|
| Полная цена | 790×$0.0030 = $2.37 | 110×$0.0375 = $4.13 | **≈ $6.50** |
| + Batch −50% | $1.19 | $2.06 | **≈ $3.25** |
| + Batch + кэш чтения | ~$1.13 | ~$2.04 | **≈ $3.17** |

> **Это и есть главное число: ~$3–7 в месяц** (см. вывод §5).

### Сценарий 3 — худший случай: полная пере-обработка ВСЕГО каждый месяц (300 A + 40 B/мес)

(перезапуск enrich на всех ~340 карточках каждый месяц — обычно не нужно, COALESCE-промоут и
`needsEnrich`/`enriched_at` пропускают уже обогащённые, см. `enrich.ts`)

| Режим | **ИТОГО $/мес** |
|---|---:|
| Полная цена | **≈ $2.40** |
| + Batch −50% | **≈ $1.20** |

> Даже «всё заново каждый месяц» — это **~$1–2.40/мес**, потому что объём (340) маленький.

## 5. Вывод по стоимости

**Steady-state месячный счёт по API — однозначные доллары (single-digit $/month).** Почему:

1. **Объём крошечный.** Это не лента из тысяч постов в день — после dedup это десятки
   уникальных событий в день (~900/мес). Стоимость линейна по объёму, а объём мал.
2. **Tiering (T139).** ~88% карточек идут по дешёвой haiku-дорожке (~$0.003), и лишь ~12%
   требуют дорогого sonnet-заземления (~$0.038). Средняя карточка ≈ **$0.0072**.
3. **Batch −50%** срезает счёт вдвое, а enrich по своей природе асинхронный (fail-soft,
   не на горячем пути `runPipeline` — см. комментарий в `enrich.ts`), так что batch подходит идеально.

Ориентир: **~$6.50/мес полной ценой → ~$3.25/мес с Batch.** Разовый bake ≈ **$2**.
А при движке по умолчанию (`claude -p`, подписка) — **$0 маржинально** (входит в подписку).

## 6. Контекст: схема опроса источников (крон) — ответ на T165

Стоимость enrichment напрямую зависит от того, **сколько новых карточек** появляется, а это
задаёт адаптивный диспетчер (T011–T015, `lib/pipeline/dispatcher.ts` + `cadence.ts`):

- **Один тик крона `*/15`** (GitHub Actions, `scheduler.yml`) → `POST /api/cron/dispatch` →
  `selectDueSources()` опрашивает ТОЛЬКО источники, у которых `next_due_at` наступил.
- **Интервал на источник** = `computePollInterval` = `clamp(observed_gap × 0.33, min, max)`;
  back-off **×1.5** когда выдача не изменилась, **×2** при ошибке.
- **Conditional-GET** (ETag / If-Modified-Since) → `304` коротит в `source_runs.not_modified`
  → **нет normalize, нет AI** (а значит и нет затрат на обогащение).
- **Дефолты по типу:** telegram 10 мин–3 ч, web 1 ч–24 ч, ticketing/api 2 ч–24 ч.

Итог: быстрый Telegram-канал проверяется ~раз в 10–30 мин, статичная страница площадки
оседает к ~раз/сутки — автоматически, без ручной настройки крона на каждый источник. Это
держит поток новых карточек (а с ним и счёт за AI) низким.

## 7. Вариант «prod без AI» (рекомендуемая форма) — детально

**Запрос пользователя:** prod не платит за AI; владелец периодически обогащает ЛОКАЛЬНО на
подписке. Это вариант **(1)** из T162.

### Как это работает

```
        ЛОКАЛЬНО (у владельца, подписка Claude, $0 AI)        PROD (Vercel, $0 AI)
  ┌───────────────────────────────────────────┐      ┌──────────────────────────┐
  │ 1. live-ingest + normalize в локальную БД  │      │  ships PRE-ENRICHED seed │
  │ 2. claude -p enrich (haiku/sonnet)         │      │  (data/seed/*.json)      │
  │ 3. npm run export-seed --commit            │ ───► │  incremental ingest ONLY │
  │    → data/seed/*.json (с title_ru, гео)    │ push │  runPipeline() БЕЗ       │
  │ 4. git push → Vercel redeploy              │deploy│  enrichClient            │
  └───────────────────────────────────────────┘      └──────────────────────────┘
```

На prod `runPipeline()` вызывается **без** `enrichClient` (см. `run.ts`: enrich пропускается,
если клиент не инжектирован) → новые карточки на prod остаются непереведёнными до следующего
локального прогона, но **сайт всё равно рендерится из БД** (constitution: deterministic from DB),
а seed уже полностью обогащён.

### Варианты триггера для локального обогащения

| Вариант | Как | Свежесть | Стоимость на prod |
|---|---|---|---|
| **Вручную** | `npm run pipeline:run` (локально) + `npm run export-seed -- --commit` + `git push` | по требованию владельца | **$0** |
| **Локальный cron** | launchd/cron на машине владельца раз в N дней запускает тот же скрипт | лаг = период крона | **$0** |
| **GitHub Action (SDK-путь)** | CI-джоб с `ANTHROPIC_API_KEY` гоняет SDK-`EnrichClient` против prod Neon БД (или регенерит дельты seed) | близко к live | платит токены (см. §4: ~$3–7/мес), но **сервер всё равно $0** |

### Компромисс

- **Плюс:** **$0 AI на prod**, никакого секрета на сервере, никакого риска перерасхода. Подписка
  уже оплачена — маржинальная стоимость обогащения $0.
- **Минус:** **лаг свежести** — новые события появляются на сайте недатированными/без RU-перевода,
  пока владелец не прогонит локальный bake. Для афиши это приемлемо (события анонсируются заранее).

### Перекрёстная ссылка на T162 (авторизация enrich в prod)

- **Вариант (1) — без prod-enrich (этот раздел, по умолчанию для запуска).** Seed предобогащён,
  prod гоняет только инкрементальный ingest. Ноль секретов. `/api/health` показывает «enrich
  stalled (info)» — информационно, никогда не валит здоровье.
- **Вариант (2) — SDK на prod.** `@anthropic-ai/sdk` + `ANTHROPIC_API_KEY` как env-переменная
  Vercel; `createSdkEnrichClient` инжектится в enrich cron/route. Работает на serverless, стоит
  токены (§4: ~$3–7/мес). Включить, когда нужна live-обработка новых карточек на prod.
- **Вариант (3) — off-prod воркер.** Маленький always-on хост / CI-джоб гоняет `claude -p` против
  prod-БД. Сохраняет подписку (без API-ключа), но нужен хост.

**Рекомендация:** запускаться с **(1)** (предобогащённый seed + инкремент, $0 AI на сервере);
добавить **(2)** (SDK + API-ключ, ~$3–7/мес), когда понадобится живое обогащение новых карточек
прямо на prod. Локальный `claude -p` flow остаётся для тяжёлого bake в любом случае.
