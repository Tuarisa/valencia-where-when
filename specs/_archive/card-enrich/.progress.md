## Learnings

- **A worked enrichment already exists in the seed** (`data/seed/events.json` id=106): `enrichment_json` with `{title_ru, description_ru, dates, venue, price, links[], confidence:0.95, notes}` — the `notes` field literally flags a Telegram-vs-Roig-Arena date conflict (21 июля vs 21 июня). This IS the schema + the anti-hallucination posture; adopt verbatim.
- **Anthropic web tools verified**: `web_search` ($10/1k searches, citations ALWAYS on) + `web_fetch` (FREE/token-only, citations opt-in). web_fetch can only fetch URLs already in context → seed the card's own `source_url`/`links_json` so it can fetch the original source deterministically.
- **Gotcha**: server tools (web_search/fetch) + `output_config.format` don't compose in one turn (server loop → `pause_turn`); and citations + structured output = **400**. Use two-turn (tool turn → constrained parse turn) or a strict custom tool as the writer.
- **events has NO `maps_url` column** (`db/schema.sql`); only `places.maps_url` exists. Maps URL for events → put in `links_json` (renders today via LinkButtons) or add a column. geo.ts already reads `places.maps_url` + `events.venue_url` as geocode sources.
- **Geo reconciliation**: enrich PROPOSES clean venue_name/address (+maps_url); existing `geo.ts` GEOCODES. Run geoEnrich AFTER enrich — more rows resolve because enrich filled the null venue/address. Don't duplicate Nominatim in enrich.
- **Coverage near-zero** (1/342 events enriched, 34 have lat, 179 have image_url) → first run is a big batch; per-card Vercel Workflow step (vercel-workflow D-WF1) for fail-soft + escaping 300s.
- **Render already consumes** title_ru/description_ru/links_json/lat/lng/image_url (detail page + Home/map) — enrich outputs flow to UI with no render change.
- **Model tiering**: Haiku 4.5 default ($1/$5), escalate to Opus 4.8 ($5/$25) for poster-OCR / low-confidence / conflict cards. No `temperature` on 4.8 (400) — use `effort:"low"` + constrained decoding for determinism.
