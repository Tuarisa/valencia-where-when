## Learnings

- Enrichment (`lib/pipeline/enrich.ts`, T043), `notify.ts`, `/api/cron/enrich` are NOT built yet; zero `claude -p`/`@anthropic-ai/sdk` usage in repo → clean slate to pick the mechanism.
- Local `claude` CLI v2.1.177 already supports `--json-schema`, `--output-format json|stream-json`, `--model`, `--fallback-model`, `--bare`. BUT `--json-schema` is POST-HOC validation (`.structured_output`), NOT constrained decoding.
- Anthropic SDK structured outputs (`messages.parse()` + `output_config.format`) ARE constrained decoding (grammar-level guarantee), GA, no beta header → strictly better than CLI for schema-guaranteed extraction. Chosen mechanism.
- JSON-schema constraints unsupported: `minimum/maximum`, `minLength/maxLength`, recursive; `additionalProperties:false` required on every object → encode date/length bounds in TEST assertions, not the model schema.
- Repo test pattern = pure-logic `.mjs` mirrors of TS + INJECTABLE `exec` (see `dedup({exec=sql})`). Mirror this for `enrich({client})` so the SDK call is mockable/offline-testable. No test framework, no `.github/workflows`, no Makefile.
- promptfoo is Node/TS-native: Anthropic provider `anthropic:messages:<model>`, assertions `is-json|javascript|llm-rubric|factuality`; `llm-rubric` auto-uses Anthropic grader when `ANTHROPIC_API_KEY` set; CAN wrap the `claude` CLI via custom-script (`exec:`) provider. Chosen primary eval tool.
- deepeval (GEval LLM-judge) = Python → toolchain mismatch; Braintrust = SaaS+cost → both deferred for a single-household app.
- AUTH/CI caveat: SDK + promptfoo `llm-rubric` need `ANTHROPIC_API_KEY` (absent in current shell); `claude --bare` also needs the key (skips OAuth). → default `npm test` MUST mock the LLM (no key/network/deterministic); live eval is `npm run eval`, key-gated + cost-gated (Haiku grader).
- No `temperature` knob on Opus 4.7/4.8 (400) → determinism via constrained decoding + low `effort`, use threshold (not exact) judge asserts.
