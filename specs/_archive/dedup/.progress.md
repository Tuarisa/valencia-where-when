## Learnings

- **`dedup.ts` exists but is NOT wired** into `run.ts` or `/api/cron/refresh` (001 T025 ☐) → dead code today. It's post-normalization, **events-only**, and never links `source_item_id` → does NOT satisfy "из исходников".
- **Good parts to keep**: pure helpers `titleSignature` (token-sort sig), `dedupKey=sig|city`, `groupByKey` (date-window sub-split), `chooseSurvivor` (weight,score), `mergeGroup`, injectable `exec`. Gap is *what it reads/links*, not *how it groups*.
- **Convention split (must reconcile):** seed uses `status='duplicate'` + `metadata.merged_into` + `normalized_from` (90 rows, Python proto); TS code uses `status='merged'` + `metadata.sources[]`. Two conventions exist — pick one, migrate.
- **Komissarenko 3→1 (ids 2/106/110):** dates conflict by a MONTH (06-21 vs 07-21) + one null → current ±3d window will NOT merge them. Needs deterministic strong-match on shared `worldafisha/persons/slava-komissarenko` URL = the "из исходников" signal.
- **PLACES FALSE-MERGE TRAP (verified in seed):** ids 4/7/8 share coords `39.4697,-0.3763` (area geocode); ids 10/11/12 share `40.4168,-3.7035` = **Madrid Puerta del Sol centroid** (Nominatim fallback for unresolved goo.gl links). 6 pairs <150m purely from placeholder coords → naive geo matcher catastrophically merges distinct venues. **Geo must be corroborating-only, gated on `geo_source='map_url'`.**
- **Raw layer already carries source signals**: `source_items` has `dedup_hash`(UNIQUE)/`external_id`/`url`/`published_at`/`raw_*`. Ingest already dedups raw items via `sourceItemHash`. Missing: propagate `source_item_id` into entity merge + match on raw url/external_id. `events`/`places` have only ONE `source_item_id` col → can't record all contributors.
- **Constitution V fixes 001 T025**: canonical order is `ingest → normalize → **dedup** → score → tag → geo → enrich` (dedup AFTER normalize, NOT "between tag and geo" as T025 said). Running before enrich = cost lever (card-enrich is $$, 1/342 enriched).
- **Decided architecture**: additive **`entity_sources` link table** (entity_type, entity_id, source_item_id, match_method, match_score) records EVERY contributing raw item (constitution I literal). Survivor.id = stable canonical id (idempotent). Deterministic strong-match (url/external_id/maps_url) + fuzzy (pure-TS Jaro-Winkler + gated haversine). No heavy deps (libpostal/dedupe are off-stack). LLM tie-break optional/flagged only.
- **Places multi-source**: `recommended_by` is a single string today — needs to become a union (mirror events `sources[]`) in `metadata_json.recommended_by[]`.
- **repeateable-events constraint**: dedup must NOT collapse series occurrences (Hemisfèric 104=11×14). Their model = `event_series`+`event_occurrences`. Komissarenko person-link must yield *related/series*, not hard merge.
