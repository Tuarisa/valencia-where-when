## Learnings

- Seed has **104 Hemisfèric `events` rows = only 11 distinct films × 14 days** (e.g. "Oceans" 14 rows). Verified `distinct_dedup_hash == rows == dates` per title.
- Root cause: `normalizers/hemisferic.ts` puts `start_date` into `eventHash` → one row per (film, day). `util.ts:eventHash` includes the date.
- `ingest.ts:parseHemisferic` already fans out per-day (loops 14 days × API), so raw `source_items` are per-(film,day) with `raw_json.session_times[]` — recoverable for regen.
- **UI collapse is render-time ONLY**: `Home.tsx` calendar groups Hemisfèric per-day into a `<details>` "N сеансов"; the FEED renders every row as its own card (no feed collapse). `is_hemisferic` = hardcoded `source==='api:hemisferic'`.
- Spec 001 FR-010 only collapses **per DAY**; THIS spec must do **per FILM (series)** — supersedes FR-010/T009.
- `dedup.ts groupByKey` deliberately SPLITS far-apart dates into sub-groups → it will NOT collapse a series; recurrence is orthogonal to cross-source dedup. Dedup must NOT scan occurrences.
- Decided model: additive **`event_series` (the card, holds enrich/score/notify) + `event_occurrences` (calendar rows)**. RRULE kept as optional metadata; occurrences are the truth (RFC 5545 hybrid). Chosen over RRULE-only (Hemisfèric schedule is irregular) and over `series_id`-on-events (leaves N enrichable rows).
- Enrich win: select target `events → event_series` → 104→11 enrich calls (card-enrich $10/1k web-search cost).
- Keep single-shot events in `events`; only recurring sources populate series (smaller blast radius).
